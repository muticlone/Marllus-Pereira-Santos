Docker

O Docker é uma plataforma poderosa de virtualização de contêineres que simplifica significativamente o desenvolvimento, implantação e execução de aplicativos em ambientes isolados. Este projeto específico utiliza contêineres para gerenciar o banco de dados MySQL, servir conteúdo com o Nginx e conter as dependências do projeto, como Laravel, PHP e Composer. Essa abordagem simplifica o processo de configuração e execução desses componentes de forma coesa.

Ao configurar este projeto, você pode facilmente implementá-lo em um servidor online seguindo alguns passos simples. A versatilidade do Docker permite que você execute o projeto em uma máquina com Linux ou Windows, tornando-o altamente acessível. Aqui está um guia passo a passo para facilitar a implementação:

Instale o Docker:

Certifique-se de ter o Docker instalado em sua máquina. Você pode encontrar instruções detalhadas no site oficial do Docker para instalação em sistemas Linux ou Windows.

Configuração do Projeto:

Clone o repositório do projeto em sua máquina. Personalize o arquivo de configuração docker-compose.yml conforme necessário, ajustando as configurações do MySQL, Nginx e do contêiner de aplicativos.

Inicie os Contêineres:

Execute o comando docker-compose up -d para iniciar os contêineres em segundo plano. Isso criará e executará os contêineres de acordo com as configurações fornecidas no arquivo docker-compose.yml.

Instale as Dependências do Projeto:

Acesse o contêiner de aplicativos com docker-compose exec app bash. No contêiner, execute composer install para instalar as dependências do Laravel utilizando o Composer.

Gere a Chave de Aplicativo:

Ainda dentro do contêiner de aplicativos, execute php artisan key:generate para gerar a chave única da aplicação.

Acesse o Projeto:

Após concluir as etapas acima, seu projeto Laravel estará rodando. Acesse-o no navegador utilizando o endereço e a porta configurados, por exemplo, http://localhost:8989.

Este processo simplificado usando Docker permite que você tenha um ambiente consistente e isolado para o desenvolvimento e implantação do projeto. Ao seguir esses passos, você garante uma implementação rápida e eficiente do seu aplicativo, independentemente do ambiente de execução.

O Laravel utiliza rotas no arquivo web.php e controladores para a navegação. Para criar um controlador, utilizamos o comando:

bash php artisan make:controller NomeDoController

Em seguida, definimos uma rota no arquivo web.php, indicando o método do controlador a ser chamado:

Route::get('/cadastrar/empresa', [CadastroEmpresaController::class, 'create']) ->name('pag.cadastrar.empresa') ->middleware('auth', 'company_or_root'); public function create(Request $request, cadastro_de_empresa $Cadastro_empresa){ $data = $request->all(); $Cadastro_empresa->create($data); return redirect('/dashboard')->with('msg', 'Empresa cadastrada com sucesso!'); }

Ao acessar a rota, o Laravel se comunica com o controlador, neste caso, o CadastroEmpresaController, e chama o método create. O método create é responsável por receber os dados provenientes da interface do usuário (frontend), encapsulados no objeto $request. Utilizando esses dados, o método cria uma nova instância de empresa no banco de dados por meio da instância do modelo CadastroDeEmpresa, que representa a tabela de empresas. Após a obtenção dos dados, o método invoca a função create do modelo CadastroDeEmpresa, que insere os dados na tabela correspondente no banco de dados. Em seguida, a função redireciona o usuário para a página de dashboard (/dashboard).

Além disso, utilizamos duas APIs externas: a API de busca de CEP e a API da Receita Federal. Essas APIs são empregadas para coletar dados das empresas e realizar o preenchimento automático de inputs nos cadastros de empresa, incluindo informações como razão social, nome fantasia e endereço.

Para o sistema de cadastro de usuários, login e controle de rotas, utilizamos o Livewire. Isso garante que os usuários possam acessar apenas as rotas predefinidas, permitindo o acesso somente quando estiverem logados, enquanto outras rotas permanecem públicas.

O sistema de login Livewire oferece uma experiência dinâmica e interativa, proporcionando aos usuários uma entrada fácil e segura ao combinar a potência do Laravel com a eficiência do Livewire para uma autenticação eficaz e sem complicações.

React é uma biblioteca JavaScript utilizada para construir interfaces de usuário interativas e eficientes.

No servidor, utilizei o Express, um framework de aplicativo web para Node.js que simplifica a criação de servidores HTTP, facilitando o desenvolvimento de APIs e aplicações web. Além disso, empreguei o nodemon para atualização em tempo real, cros e bodyParser para simplificar o processamento de dados provenientes do corpo (body) de requisições HTTP. No banco de dados, utilizei o MySQL com Sequelize que simplifica a interação com bancos de dados relacionais, fornecendo uma interface em JavaScript para realizar operações de banco de dados de maneira mais intuitiva.

                        app.post("/cadastrar/cliente", async (req, res) => {
                            try {
                                const { dados } = req.body;
                        
                                if (dados.cpf) {
                                    const clienteExistente = await Cliente.findOne({
                                        where: { cpf: dados.cpf },
                                    });
                        
                                    if (clienteExistente) {
                             return res.status(400).json({ msgErro: "Cliente já cadastrado com este CPF" });
                                    }
                                }
                        
                                const novoCliente = await Cliente.create({
                                    nome: dados.nome,
                                    sobrenome: dados.sobrenome,
                                    // Adicione outros campos conforme necessário
                                });
                        
                                res.json({ mensagem: "Novo cliente cadastrado com sucesso!!" });
                            } catch (error) {
                                console.error("Erro ao cadastrar cliente:", error);
                                res.status(500).json({ msgErro: "Erro interno do servidor" });
                            }
                        });

No servidor, req.body é utilizado para receber os dados. Em seguida, verifica-se se já existe outro cliente cadastrado com o mesmo CPF. Se existir, envia uma resposta para o frontend; se não existir, realiza o cadastro no banco usando o Sequelize e envia a resposta para o frontend.
